## フォルダ構成（ルール）

```
order-made-designs/
├── README.md                 ← このファイル（全体の目次）
├── airtag-case/
│   ├── README.md             ← 商品ごとの説明
│   ├── 3d/                   ← 3Dプリント用（.stl .3mf .step）
│   ├── laser/                ← レーザー加工用（.svg .dxf）
│   └── cut/                  ← 裁断用（.dxf .pdf）
├── number-keyholder/
│   ├── README.md
│   ├── 3d/
│   ├── laser/
│   └── cut/
└── club-keyholder/
    ├── README.md
    ├── 3d/
    ├── laser/
    └── cut/
```

### ファイル名のルール

```
商品名-用途-v1.拡張子

例：
airtag-case-v1.stl
airtag-case-v1.svg
number-keyholder-v1.dxf
```

- **英小文字 + ハイフン** だけ使う（スペース・日本語・記号は避ける）
- 改良したら **v2, v3…** に上げる（古い版は消さず残す）

---

## 新しい商品を追加するとき

1. 商品フォルダを作る（例：`number-keyholder/`）
2. 中に `3d/` `laser/` `cut/` と `README.md` を作る
3. この README の **商品一覧表** に1行追加
4. ファイルを入れて GitHub にアップロード

---

## 載せないもの

- 顧客の個人情報・名入れデータ
- 有料素材のパターン（ライセンス違反）
- AWS の鍵ファイル（.pem 等）
